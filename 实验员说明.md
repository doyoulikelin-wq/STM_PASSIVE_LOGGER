# STM 实验记录 / 标注工具 — 实验员说明书

> 这份文档**不需要懂编程**就能看懂。从头到尾照做即可。
> 所有界面操作都通过 **双击 .bat 文件** 完成；命令行黑窗口只用来按 `Ctrl+C` 退出。
> 出问题时，把黑窗口里的红色英文报错截屏发给开发者就行。

---

## 这个工具是干什么的

实验过程中：
- **被动地** 在后台读 Nanonis 的当前状态（Bias / Current / Z / 反馈开关 / 扫描状态）。
- 监听 Nanonis 的保存目录，每当你 Save 一张 `.sxm` 扫描，就自动把它收进数据库并生成预览图。
- 把信号、扫描、事件全部统一存到一个 SQLite 数据库 + JSONL 备份。
- **不会** 给 Nanonis 下任何控制命令，**不会** 干扰你正常实验。

实验后（同一台电脑或拷到任何一台电脑）：
- 打开浏览器界面，给每一张扫描打 5 维标签 + 自由备注。
- 第二个人可以做 review，标 accept / dispute / need_redo。

## 1. 适配的软件版本

| 必备项 | 版本 | 说明 |
|---|---|---|
| **Windows 10 / 11 (64-bit)** | — | 主要测试平台 |
| **Nanonis Mimea V5e** | — | 必须打开 TCP Programming Interface |

- `numpy`、`PyYAML`、`matplotlib`、`pillow`、`nanonis-spm`

> 标注 UI 用 Python 标准库的 `http.server`, 不需要装 Flask / FastAPI。

## 2. 第一次使用 (每台电脑只做一次)

1. 把整个 `stm_logger_v0.1.zip` 解压到一个 **英文路径**下, 例如 `D:\stm_logger\`
   - 路径里不要有中文 / 空格 / 特殊符号。
   - **一定要完整解压** (不能双击 zip 直接看里面的 .bat 运行, 会报错)。
2. **双击 `自检.bat`**: 会弹一个黑窗口, 检查内置 Python 和依赖是否完整。
   - 看到 `自检通过!` 就可以了。
   - 自检还会检查标注 UI 页面文件和标签配置是否完整。
   - 报错就把黑窗口截图发给开发者。

> 之后换台电脑, 直接把整个文件夹拷走就能用, 不用重新装。

---

## 3. Nanonis 这边要做什么（每次实验前）

1. 打开 **Nanonis 主程序**。
2. 打开 **TCP Programming Interface**（菜单 `Utilities → TCP Programming Interface`）。
3. 在 **`File / Session`** 面板里，确认 **保存路径**已经设到一个**专用目录**（例如 `D:\STM\S2`）。
   - 这个路径会被 logger **自动读出来**，你不用告诉它。
   - 如果以前从来没设过，logger 会跳过扫描捕获，只采信号。

> 端口号你不用关心：logger 会自动从 6501 → 6502 → 6503 → 65004 依次试，谁应答用谁。

## 4. 启动一次实验记录

### 4.1 先做健康检查（推荐，30 秒）

**双击 `健康检查.bat`**，看到类似下面的输出就说明 Nanonis 通了：

```
{
  "connected_port": 6502,
  "session_path": "D:\\STM\\S2",
  "sample_snapshot": { "bias_V": 2.0, "current_A": -6.5e-11, ... },
  "ok": true
}
```

如果 `"ok": false` 或者最下面写了 `Could not reach a Nanonis Programming Interface`，说明 Nanonis 的 TCP 接口没开 → 回到第 3 节。

如果最外层是 `"ok": true`, 但 `"version": {"ok": false, ...}`，通常也可以继续使用：有些 Nanonis 版本的 `Util.VersionGet` 会超时，但 Bias / Current / SessionPath 等核心读数仍然正常。以 `port_probe_results` 里的 `core_probe` 为准，只要 `Bias_Get` 和 `Current_Get` 是 `ok`，就说明不是“只连上端口”，而是真的能交互数据。

### 4.2 正式开始记录

**双击 `启动logger.bat`**，按提示输入：

| 提示 | 填什么 | 必填 |
|---|---|---|
| operator | 你的姓名（拼音/英文，不要空格） | ✅ |
| sample | 样品编号，如 `BP5-001` | ✅ |
| tip | 针尖编号，如 `tip01` | ✅ |
| material | 材料，如 `Bi2Se3` | 选填 |
| 备注 | 任何文字 | 选填 |

回车之后会出现：
```
[stm-logger] connected to Nanonis on 127.0.0.1:6502
[stm-logger] auto-discovered watch dir from Nanonis: D:\STM\S2
[stm-logger] session started: 20260521_103045_BP5-001_tip01
[stm-logger] press Ctrl+C to stop.
```

**看到 `session started` = 已经在记录**。这个黑窗口要**一直保留**（最小化没关系）。

### 4.3 怎么确认它确实在工作

任选一种验证方式：

- **看黑窗口**：每秒会有一行类似 `signals batch 50 written` 的日志。
- **双击 `查看数据.bat`** 打开数据文件夹，进去 `sessions/<你的session_id>/`，应该看到 `signals.jsonl` 在不停变大。
- 在 Nanonis 里 **Save 一张扫描**，几秒后看 `data/previews/<session_id>/` 应该出现一张 `.png` 预览图，黑窗口里也会有 `scan_captured` 字样。

### 4.4 怎么退出

**回到那个黑窗口，按一次 `Ctrl + C`**（不是关 X）。会看到：
```
Received signal 2, shutting down
[stm-logger] session stopped: 20260521_103045_BP5-001_tip01
```

按任意键关掉窗口即可。`Ctrl+C` 会让它先把缓存写完、把 `session_end` 事件写进数据库，**安全**。

> **千万不要直接点窗口的 X 关掉**，那样最后一批数据可能丢失。
> 万一直接 X 了：JSONL 仍然完好（每秒落盘），只是 SQLite 里 `end_ts` 没更新——不影响数据本身。

## 5. 数据放在哪里

**所有数据**都在你解压目录下的 `data\` 文件夹（双击 `查看数据.bat` 直接打开）：

```
data\
├── session.sqlite              ← 主数据库, 用 DB Browser for SQLite 可直接查看
├── sessions\
│   └── 20260521_103045_BP5-001_tip01\
│       ├── events.jsonl        每个事件一行, 文本可读
│       ├── signals.jsonl       每次轮询一行
│       └── scans.jsonl         每张新扫描一行
└── previews\
    └── 20260521_103045_BP5-001_tip01\
        └── <scan_id>_Z_fwd.png    扫描预览 (Z 通道优先, 否则 Current / LI)
```

`session_id` 格式 = `年月日_时分秒_样品_针尖`，全局唯一。

> **拷数据**给同事 / 备份：直接把整个 `data\` 文件夹复制走就行，**没有任何数据库锁文件依赖**。

## 6. 自动触发什么 / 什么会被记录

logger **不要求**你做任何特别的操作；以下事情会**自动**被记录：

| 你的操作 | logger 自动记什么 |
|---|---|
| 启动 logger | 写入 `sessions` 表一行 + `session_start` 事件 |
| 一直开着 logger | 每秒把 Bias / Current / Z / 反馈开关 / 扫描状态写入 `signals` 表 + `signals.jsonl` |
| 在 Nanonis 里点 **Save** 扫描（落盘 `.sxm`） | 解析 `.sxm` 元数据 → `scans` 表插一行 + 生成 PNG 预览 + 写 `scan_captured` 事件 |
| 临时改 bias / 反馈 / 扫描参数 | **数值会出现在下一条 `signals` 记录里**（不写额外事件） |
| 按 `Ctrl+C` | 写 `session_end` 事件（含 signals/scans 计数）+ 更新 `sessions.end_ts` |

> logger **不知道**「你刚才框选了一个新扫描区域」这种 GUI 动作；它只看 Nanonis 的状态值变化和 `.sxm` 落盘。所以**框扫描后请一定要 Save**，否则那次扫描不会进数据库。

## 7. 实验完之后做标注

可以**离线**完成，不需要 Nanonis、不需要再去 STM 实验室。把整个目录拷到任何一台装好本工具的电脑都行（数据是 `data\` 文件夹）。

### 7.1 启动标注界面

**双击 `打开标注UI.bat`**：浏览器会自动打开 `http://127.0.0.1:8765`。

### 7.2 怎么标

1. 顶部 **「标注者」** 输入框填你的姓名（这会写进 `labels.annotator` 字段；浏览器会记住）。
2. 默认显示「**仅我未标**」的扫描列表，按时间倒序。可以按 session 过滤。
3. 左侧点一张扫描 → 右侧出现：
   - 上方：扫描的元数据 + PNG 预览图
   - 中间：表单（5 维标签下拉 + 复选框 + 数值评分）
   - 最下面有个 **`annotator_notes`** 自由文本框 → **写下你看到这张图的犹豫点 / 现场观察**，review 时同事能看到。
4. 点 **「保存我的标签」** 或 **「保存并下一张 →」**。
5. 同一张图下方会列出**其他人的标签**，每条卡片下面都有一个小表单可以打 **`accept / dispute / need_redo`** + 写 review 评论 → 这是 **review 流程**，reviewer 是顶部填的当前姓名。

### 7.3 关键性质

- **多人独立标同一张** = 各自一条记录共存（主键是 `(scan_id, annotator)`）。
- **同一人重标同一张** = 覆盖（`updated_ts` 更新）。
- 进度查询：双击运行不带参数的 PowerShell，输入下面这一行可以看进度：
  ```powershell
  python -m stm_experimenter_agent.cli annotate-stats --data-root .\data --annotator 你的名字
  ```

### 7.4 退出标注 UI

回到那个黑窗口按 **`Ctrl + C`**。浏览器标签页随便关。

## 8. 数据库里到底长什么样

数据库文件：`data\session.sqlite`。推荐用免费的 **[DB Browser for SQLite](https://sqlitebrowser.org/)** 双击打开浏览。

### 8.1 表结构速览

| 表 | 一行 = | 关键字段 |
|---|---|---|
| `sessions` | 一次实验 | `session_id` (PK), `start_ts`, `end_ts`, `operator`, `sample_id`, `tip_id`, `material`, `instrument`(JSON) |
| `scans` | 一张被保存的 .sxm 扫描 | `scan_id` (PK), `session_id`, `captured_ts`, `sxm_path`, `preview_path`, `bias_V`, `setpoint`, `pixels_x/y`, `range_x/y_m`, `offset_x/y_m`, `angle_deg`, `channels`(JSON) |
| `signals` | 一次信号轮询 (默认每秒一次) | `session_id`, `ts`, `bias_V`, `current_A`, `z_m`, `z_ctrl_on`, `scan_status`, `errors`(JSON) |
| `events` | 一个离散事件 | `session_id`, `ts`, `kind`(`session_start` / `watch_dir_resolved` / `scan_candidate_seen` / `scan_captured` / `session_end` / ...), `payload`(JSON) |
| `labels` | 一个人对一张扫描的标注 | `(scan_id, annotator)` (复合 PK), `image_quality`, `tip_state`, `surface_quality`, `artifact_tags`(JSON), `research_value_label/score`, `next_action`, `confidence`, `reason_text`, **`annotator_notes`**, **`review_status`**, **`review_comment`**, **`reviewer`**, `reviewed_ts`, `created_ts`, `updated_ts` |

### 8.2 几个常用查询（在 DB Browser 的 "Execute SQL" 标签里粘贴）

```sql
-- 最近 10 次 session
SELECT session_id, operator, sample_id, tip_id, datetime(start_ts,'unixepoch','localtime') AS start
FROM sessions ORDER BY start_ts DESC LIMIT 10;

-- 某 session 的全部扫描
SELECT scan_id, datetime(captured_ts,'unixepoch','localtime') AS t,
       bias_V, pixels_x, pixels_y, preview_path
FROM scans WHERE session_id = '20260521_103045_BP5-001_tip01'
ORDER BY captured_ts;

-- 某人已标但还没人 review 的标签
SELECT scan_id, image_quality, tip_state, annotator_notes
FROM labels WHERE annotator='myl' AND review_status IS NULL;

-- 同一张图不同人的分歧
SELECT scan_id, annotator, image_quality, tip_state, surface_quality
FROM labels WHERE scan_id IN (
  SELECT scan_id FROM labels GROUP BY scan_id HAVING COUNT(*) > 1
) ORDER BY scan_id;
```

## 9. 常见问题

| 现象 | 原因 / 处理 |
|---|---|
| 双击 `自检.bat` 报 `找不到内置 Python` | zip 没完整解压, 或解压路径含中文/空格。换成纯英文路径重新解压。 |
| 双击任何 .bat 弹一下黑窗口就闪退 | 不要把 .bat 拖出文件夹单独跑。必须保持 .bat 和 `python_runtime\` 同一目录。 |
| `健康检查.bat` 显示 `Could not reach a Nanonis Programming Interface on any of ...` | Nanonis 主程序的 TCP Programming Interface 没开。在 Nanonis 里打开 `Utilities → TCP Programming Interface`。 |
| `health check` 里 `connected_port` 不是 6501 | 正常。工具会依次检查 6501/6502/6503/6504/65004，只有 Bias / Current 等核心读数能返回的端口才会被选中。 |
| `version.ok` 是 `false` 但最外层 `ok` 是 `true` | 通常可以继续用。`Util.VersionGet` 在部分 Nanonis 上会超时；只要 `port_probe_results.core_probe` 里 Bias / Current 是 `ok`，logger 就能正常记录。 |
| `session_path` 显示 `null` | Nanonis 还没设过保存目录。在 Nanonis `File → Path` 设一个，重启 logger。 |
| `signals` 表 `errors` 字段不为空 | 单个字段读取失败（如 ZCtrl 未启用），其他字段照常入库，可忽略。 |
| `scans` 表一直没新行 | Nanonis 还没保存任何 `.sxm`，或保存路径不在 logger watching 的目录。检查健康检查里的 `session_path`。 |
| 预览图打不开 | `.sxm` 不含 Z / Current / LI_Demod_1_X 任一通道，或 matplotlib 渲染失败。看黑窗口日志。 |
| 标注 UI 浏览器一直转 | `打开标注UI.bat` 的黑窗口必须开着。或者地址错了，应该是 `http://127.0.0.1:8765`。 |
| 一台电脑想跑两个 logger | 不要这么做。一份 `data\` 目录同一时间只允许一个 logger 写入。 |
| 想发数据给同事看 | 把 `data\` 整个文件夹打包发就行，对方只要装好本工具，把 `data\` 放在一起就能继续标注。 |
| 出现 NI 的 `nimdnsNSP.dll 被阻止加载到本地安全机构` 弹窗 | **跟本工具无关**。是 Windows LSA 拦了 NI 自己的 mDNS 服务，可勾选「不再显示」忽略。本工具走 TCP 直连，不依赖该 DLL。 |

## 10. 一行不变的「安全承诺」

代码里有**两层白名单**保护：
1. driver 层只允许调用 5 个只读方法（Bias / Current / Z / 反馈状态 / 扫描状态 / 版本 / Session 路径）。
2. raw protocol 命令也走白名单。

任何 `*_Set / Pulse / Approach / Motor / TipShaper / Withdraw / AutoApproach` 调用都会立即抛 `WriteOperationNotAllowed` 异常。

→ logger 不可能误操作仪器。
